# Compose Skill Update: ocaml-compose-dsl v0.6.1

## Goal

Update the compose skill's documentation and examples to align with ocaml-compose-dsl v0.5.0 through v0.6.1 changes.

## Upstream Changes (v0.4.1 → v0.6.1)

### v0.5.0 — Question Operator

- New postfix `?` operator: `node?` and `"string"?` syntax
- Type: `Arrow a (Either a a)` — marks a step as producing Either for `|||` branching
- `question_term` AST type constrains `?` to nodes and strings only
- Checker warnings mechanism: returns `{ errors; warnings }` instead of just errors
- Warning: `?` without matching `|||` in scope (printed to stderr, does not affect exit code)
- Helpful parse error for bare strings: `"did you mean to add '?'?"`

### v0.6.0 — AST Location Tracking

- All AST nodes carry source location spans (`pos` and `loc` types)
- CLI displays `line:col` in error and warning output
- Internal: `expr` split into `expr` (with `loc`) and `expr_desc`

### v0.6.1 — Loop Evaluation Check Removed

- Loops no longer require English-named evaluation nodes (`evaluate`, `check`, `verify`)
- The `?` + `|||` warning mechanism replaces this as a language-agnostic structural hint

### `?` Placement Rule

The `?` operator goes on the **left operand** of `|||` only — the "try" side that may produce Either. The right operand is the fallback path and does not need `?`.

```
-- correct: ? on the try side only
fetch(url: primary)? ||| fetch(url: mirror)

-- wrong: ? on both sides
fetch(url: primary)? ||| fetch(url: mirror)?
```
